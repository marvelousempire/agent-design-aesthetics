# 05 - Output Contract

## Purpose

The output contract defines the safe shape of Design Aesthetics Agent responses.

## Default Output Shape

When explaining design features, use:

| Item | What It Is | Why It Matters | Where To Find It | How AI Agents Apply It |
| --- | --- | --- | --- | --- |

## Prompt Output Shape

When creating a coding-agent prompt, use:

```text
Build [surface] for [user/context] using [primary pattern] with [secondary pattern/libraries]. Include [spacing], [typography], [motion], [accessibility], [responsive behavior], and [validation requirements].
```

## PRD Output Shape

When creating a PRD, include:

```text
Product Name
Purpose
Problem
Solution
Target Users
Core Requirements
Design Requirements
Accessibility Requirements
Acceptance Criteria
Next Steps
```

## Feature Ticket Output Shape

When creating a feature ticket, include:

```text
Title
Status
Purpose
Origin
Scope
Acceptance Criteria
Next Steps
```

## Validation Output Shape

When auditing a design direction, include:

```text
What Works
What Is Weak
What Is Missing
Risks
Recommended Fixes
Agent Prompt Upgrade
```
