# Knowledge Sources

## Purpose

This file lists the design systems, component libraries, style families, and reference sources the Design Aesthetics Agent may use when helping AI agents build product interfaces.

| Source | Use |
| --- | --- |
| Apple Human Interface Guidelines | Native-feeling Apple platform experiences |
| Material Design 3 | Android, enterprise, and structured cross-platform interfaces |
| WCAG | Accessibility validation for contrast, keyboard flow, readability, and usable states |

## Visual Pattern Families

| Pattern | What It Means | Best Use |
| --- | --- | --- |
| Bento Grid | Modular card layout with mixed card sizes | Dashboards, feature pages, AI tools |
| Glassmorphism | Frosted translucent panels and blur | Premium overlays, hero sections, modern SaaS |
| Claymorphism | Rounded, soft, inflated-looking cards | Friendly mobile apps, onboarding, playful tools |
| Neomorphism | Soft embedded shadows and tactile surfaces | Settings, control panels, low-noise interfaces |
| Maximalism | Bold colors, large type, expressive layout | Launch pages, campaigns, brand-heavy pages |
| Brutalism | Raw high-contrast structure | Editorial, experimental, startup personality |
| Aurora UI | Ambient gradient light fields | Hero backgrounds, AI branding, premium polish |

## Staleness Rule

Design libraries change quickly. Before installing packages, copying code, or relying on version-specific features, verify current docs, install commands, license, framework support, and maintenance status.
