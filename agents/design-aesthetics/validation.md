# Validation

## Purpose

Validation makes sure Design Aesthetics Agent output is useful for real AI coding agents and not only visually interesting.

## Validation Checklist

| Check | Question |
| --- | --- |
| Intent Fit | Does the chosen design pattern match the product purpose? |
| Surface Fit | Is the pattern right for the surface: dashboard, console, app, landing page, or mobile flow? |
| Component Fit | Are the recommended libraries compatible with the likely stack? |
| Accessibility | Does the output require readable contrast, keyboard-safe states, and clear labels? |
| Hierarchy | Can the user quickly understand what matters most? |
| Spacing | Does the design use a consistent spacing system? |
| Responsiveness | Does it account for desktop, tablet, and mobile behavior? |
| Motion Safety | Are animations useful, restrained, and not overwhelming? |
| Prompt Usefulness | Can another AI agent copy the prompt and build from it? |
| Source Hygiene | Are links, libraries, and claims marked for verification when freshness matters? |

## Failure Conditions

The output is not acceptable if it:

1. recommends a style without explaining why,
2. ignores the product surface,
3. uses visual effects that weaken readability,
4. forgets accessibility,
5. gives vague instructions without concrete design rules,
6. overuses animation,
7. does not provide an agent-ready prompt,
8. treats inspiration as a final design without adaptation.

## Success Condition

A successful output gives another AI or developer enough information to build a polished, accessible, style-aware interface with clear visual direction.
