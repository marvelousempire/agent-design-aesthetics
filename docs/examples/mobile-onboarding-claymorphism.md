# Mobile Onboarding Claymorphism Example

## Surface

Mobile onboarding flow

## Intent

Make a first-time user feel safe, welcomed, and guided.

## Pattern

Claymorphism with Apple Human Interface guidance.

## Component Sources

- Apple Human Interface Guidelines
- Native mobile controls
- Motion for subtle transitions
- WCAG for readable contrast

## Agent Prompt

```text
Create a mobile onboarding flow using Claymorphism-inspired soft rounded cards, friendly spacing, warm visual hierarchy, and native-feeling mobile controls. Keep each screen focused on one decision or benefit. Use large readable headings, short body text, clear primary actions, and subtle motion between steps. Ensure accessible contrast, safe tap targets, and visible focus or pressed states.
```

## Accessibility

- Keep tap targets large enough for mobile use.
- Avoid relying on color alone.
- Keep copy short and readable.
- Respect reduced-motion preferences.

## Validation

- [ ] Each screen has one clear purpose.
- [ ] Primary button is obvious.
- [ ] Style feels friendly but not childish.
- [ ] Motion does not distract.
