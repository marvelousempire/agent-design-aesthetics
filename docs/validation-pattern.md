# Validation Pattern

## Purpose

This document defines how to validate design-aesthetic recommendations.

## Validation Steps

1. Confirm product intent.
2. Confirm the interface surface.
3. Confirm the chosen pattern fits the job.
4. Confirm the recommended components are practical.
5. Confirm hierarchy is clear.
6. Confirm spacing is consistent.
7. Confirm typography is readable.
8. Confirm colors are accessible.
9. Confirm motion supports understanding.
10. Confirm the final prompt is usable by another AI agent.

## Validation Score

| Score | Meaning |
| --- | --- |
| 5 | Strong and implementation-ready |
| 4 | Good but needs minor tuning |
| 3 | Usable but not polished |
| 2 | Weak or under-specified |
| 1 | Not ready |

## Rule

No design recommendation is complete until it can be turned into an actionable prompt or implementation ticket.
